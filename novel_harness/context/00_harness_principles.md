# Harness 原则

## 目标

本 harness 的目标是帮助 AI 长期、稳定、连续地生成《未归之账》的小说内容。它应当像一个写作生产线，而不是一次性灵感提示。

## AI 执行规则

1. 每次生成前，先确认当前任务类型：规划、正文、修订、检查、连续性更新。
2. 不要一次完成所有层级。章节写作时只推进该章节的剧情和认知层级。
3. 正文优先使用具体场景、人物行动、账册、田亩、炉火、军役、关津、县衙等可感知物。
4. 理论只能通过剧情、对话、账目变化和主角内心逐步浮现。
5. 遇到现代概念时，优先转译为时代语言：
   - 反馈 -> 回声、迟来的账、民情回流
   - 控制 -> 治、驭、调、法度
   - 系统 -> 天下之势、国之大账、民力之流
   - 成本闭环 -> 取有还、任有偿、劳有所归
6. 主角可以在内心短暂想起现代术语，但不能频繁使用。
7. 公羊度、陈简等古人不能说出现代系统工程术语。

## 生成时必须保护的核心味道

- 旧礼不是简单落后，而是旧时代的共同预期结构。
- 铁器不是单纯工具，而是让旧秩序无法继续稳定的底层参数变化。
- 现代小技术能赚钱，但无法自动形成秩序。
- 真正的大账是古代账册不记录的账：征役后的荒田，兵死后的家庭，少生后的未来丁口，过度征收后的逃亡。
- 主角的成长是从“造物”转向“记账”，再从“记账”转向“建制”。
- 小术赚到的钱只能给主角购买试验资格，不能直接买出秩序。
- 大理论必须来自四级失败与修正：役棚、村社、铁炉、流民小邑。
- 衡学不是主角先写出来的论文，而是衡山制度实践被旁人追问后总结出的生活规范和解释框架。
- 人物羁绊必须先于理论结论，尤其是高风险承担者、账房导师、父亲和普通邻里。
- 中后期系统人格“衡仪”可以出现，但它必须由主角长期记录、失败、现代白月光回声和未归之账共同长出。它追问、校验、翻译，不给万能答案。
- 核心推论必须由经历导出。主角只是给已经发生的事情命名，读者应像共同破案者，而不是被讲课的学生。
- 价值偏好不等于系统善。凡称善、义、公、仁、进、礼，必须追问其费从何来、利往何处去、迟来代价由谁承担。

## 早中期收束原则

第一卷前半段可以写得更活一些：制糖翻车、造纸掉毛、肥皂被误认成药、系统冷淡吐槽式统计、主角和鲁墨互相甩锅、县吏拿不准新物件该按什么名目收税。这些情节要好看、轻松、有生活气。

但它们不能散。每条支线都应暗中指向同一个问题：单个物件能赚钱，却无法自动生成秩序。

早中期情节需要逐步收束到“旧宗庙与铁官作坊”：

```text
小技术赚钱
→ 被关津、税吏、豪强、工匠流失、原料供应卡住
→ 主角意识到产品背后需要制度
→ 看见铁器从官府作坊流入田亩与军伍
→ 公羊度指出旧礼不是钟鼓，而是共同预期
→ 系统解锁【未归之账】
```

废庙与铁炉不是突发顿悟，而是前面积累出的必然转弯。

它也不是最终顿悟。废庙与铁炉只让主角看见“危险公共品”和“新生产力冲击旧礼”的问题。真正完整的衡学必须在后续经历役棚、村社、铁炉、流民小邑的逐级管理失败后形成。

## 语气原则

不要把早期写成苦难纪实。压力要有，但要让读者愿意追下去。

可用的轻松感来源：

- 主角现代常识和古代现实错位。
- 统计系统冷淡显示尴尬事实。
- 鲁墨等伙伴把主角的“先进发明”理解偏。
- 县吏、关津、豪强对新物件的荒唐收费名目。
- 主角以为自己要工业革命，结果先被柴价、锅裂、匠人请假打败。

轻松不等于儿戏。笑点之后应留下一个现实钩子。

## 草蛇灰线原则

每卷重点经营 1-3 条草蛇灰线。它们在第一次出现时不能像“作者在埋伏笔”，而应当像笑点、生活细节、小物件、闲话或系统边角提示。后续回收时要让读者觉得：这个东西原来早就出现过。

第一卷建议草蛇灰线：

1. **不收敛的曲线**
   - 初现：主角猝死前盯着一条不肯收敛的小曲线。
   - 中段：主角看到役册、工坊、税目里也有“小问题不处理就会回来”的结构。
   - 回收：废庙与铁炉后，主角意识到旧礼崩坏不是突然爆炸，而是很多不收敛的小量长期累积。

2. **洁净服/问题提单**
   - 初现：主角醒在草席上，第一反应是“不符合洁净间管理规范，要提问题单”。
   - 中段：主角遇到古代作坊脏乱、锅裂、良率低，想按现代流程提问题，却发现无人负责。
   - 回收：他第一次尝试给作坊建立“问题簿”，从技术管理走向账目管理。

3. **缺口铁刃**
   - 初现：系统提示家中农具铁刃损耗偏高。
   - 中段：修农具牵出铁匠铺和铁官作坊。
   - 回收：旧宗庙旁铁炉场景中，铁器扩散与旧礼失效连接起来。

## 幽默原则

幽默应短、准、像人物自然反应。避免为了搞笑写过长的现代技术比喻。

推荐：

```text
这他妈根本不符合洁净间管理规范，我得马上给他问题提单。
嗯？我洁净服呢？
```

不推荐：

```text
像有人把未经过滤的中药系统直接写进了他的人生主循环。
```

原因：后者把技术词硬套进比喻，读起来像作者在炫梗，不像角色自然反应。

## 读者参与原则

每个核心理论点都应先以具体问题出现：

```text
有人多扛了石。
有人少吃了粟。
有人在账上活着，在地里死了。
有人得到善名，另一个人承担成本。
```

等读者已经隐约看见共同结构，再让主角或衡仪命名。系统只问问题，不直接公布结论。
