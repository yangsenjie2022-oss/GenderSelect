# 读者参与推导原则

## 核心目标

读者不能感觉自己被作者按头教育，而应感觉自己和主角一起推导出了结论。

理想体验：

```text
我早就觉得这些事之间有共同结构。
主角终于把它说出来了。
```

## 写作顺序

任何核心理论第一次命名前，必须先经过：

```text
现象
→ 具体冲突
→ 局部失败
→ 读者隐约感到共同结构
→ 主角给出命名
```

不要反过来：

```text
理论
→ 举例
→ 作者解释
```

## 先给现象，不给答案

例如“善意 target 会转嫁成本”，不要先写结论。

先写：

- 主角禁止鞭打。
- 役夫短期感激。
- 工期变慢。
- 县里催工。
- 伍仓和壮劳力夜里偷偷补工。
- 真病者不敢申诉，因为假病者占了名额。
- 主角获得“我比旧役吏仁慈”的心理安慰，却发现有人沉默地多背了成本。

等读者已经感觉“不对劲”后，主角再说：

```text
我只是把鞭子从役吏手里拿走，却没看见它落到了谁背上。
```

## 让读者先站在主角这边

主角早期使用现代平等、人权、少受苦 target 时，读者应本能支持他。

然后通过真实反馈让读者和主角一起修正：

```text
价值目标不是错。
问题是它需要成本闭环、替代劳力、粮食冗余、统计核验和承诺兑现。
```

这样读者不是被骂醒，而是和主角一起看见支撑结构。

## 读者可见，主角未必立刻看见

可以让某些代价先被读者看见，但主角当时忙于工期、账册或事故没有完全意识到。

示例：

- 伍仓嘴上骂，夜里多扛两趟石。
- 沉默壮劳力把粟饼分给弟弟，自己没吃饱。
- 真病者看见假病者申诉成功，于是把话咽回去。

几章后角色或主角再指出：

```text
你以为没人挨打了？只是没人喊疼了。
```

读者会因为自己提前看见而获得参与感。

## 系统只问问题，不给结论

系统/衡仪不能像老师一样把结论喂给读者。

推荐：

```text
【当前目标：减少鞭打】
【未定义目标：工期延误由谁承担】
【病休人数：三】
【补工人数：二】
【未登记加役：一】
【是否归账？】
```

避免：

```text
【结论：善意 target 会转嫁成本】
```

## 多角色提供局部正确性

不要让主角独占真理。

同一问题应由不同角色给出半对解释：

- 伍仓：扛石头的人知道风险落在谁身上。
- 陈简：病名好写，补役难找。
- 公羊度：仁若不能久，使人信一日，疑十日。
- 申不疑：人不畏法，渠不成。
- 鲁墨：器物能做，批量和匠人未必能稳。

主角后期的理论不是推翻所有人，而是把这些局部正确性放到同一张图里。

## 总结要晚于读者直觉半拍

最佳时机：

```text
读者已经隐约看见共同结构，但还没有准确命名。
```

这时主角说一句能命名前文经验的话：

```text
账不写在册上，就写在人身上。
```

如果读者还没看懂就总结，会像说教。  
如果读者早就全懂还拖太久，会显得磨叽。

## 让理论成为读者可用的预测工具

当读者理解“未归之账”后，后续剧情应让读者能提前判断某个政策或命令哪里会回流。

例如官员提出：

```text
今年加征两成，明年修三渠，后年兵强。
```

读者应自然想：

- 谁付成本？
- 农时怎么办？
- 逃户怎么办？
- 兵偿怎么办？
- 承诺兑现了吗？

主角也问这些问题，读者会得到“我也看出来了”的爽感。

## 价值观筛选方式

作品会天然筛选部分价值观不一致的读者，但不要用作者口吻骂人。

正确方式：

- 不喊口号。
- 不审判读者。
- 不写“愚蠢的人才会这样想”。
- 让不同制度选择产生不同后果。
- 法家、儒家、墨家、道家等路线都要有局部正确性和局部代价。

价值争议通过系统后果筛选，不通过作者审判筛选。

## 问题簿结构

关键理论点可以先以问题簿形式出现，而不是答案。

示例：

```text
问题一：无人挨打后，工期为何更慢？
问题二：同食之后，谁先饿？
问题三：病者可休后，谁补空？
问题四：人人可诉后，谁还沉默？
```

中后期主角翻旧问题簿，读者也一起回忆。

这能把第一卷“问题提单”草蛇灰线升级成全书的推导结构。

## 主角也要反复说错

主角不能每次总结都对。

示例路径：

```text
第一次：果然还是需要鞭子。
第二次：那就多给粮。
第三次：按劳分就行。
第四次：不只是鞭子、粮、劳分，而是任、利、险、偿要同账。
```

读者会因为看见主角修正而相信理论不是作者硬塞。

## 禁止

- 不要用长篇旁白直接解释公理和推论。
- 不要让系统替读者总结答案。
- 不要让反对者都像蠢人或坏人。
- 不要在同一章第一次提出问题、第一次失败、第一次总结、第一次立学派。
- 不要把读者当学生，要把读者当共同破案者。
