# 风格指南

## 叙述风格

- 旁白清楚、现代、节奏感强。
- 场景有古代质感，但不要堆砌生僻字。
- 对话可以有文言味，但必须让现代读者一遍读懂。
- 不写空泛宏论。每段理论都要依附于具体人物、物件或事件。

## 推荐语气

主角内心：

```text
克制、分析、带一点现代工程人的冷幽默和挫败感。早期可以让他嘴硬、倒霉、被现实打脸，但不要油滑。
```

公羊度：

```text
沉静、古雅、少怒、多问，像旧时代最后一个知道旧礼为何有用的人。
```

陈简：

```text
朴素、冷眼、会用基层经验打脸主角。
```

申不疑：

```text
锐利、有效、冷酷，不是蠢反派。
```

## 避免

- “系统叮，恭喜宿主”式轻浮语气。
- “古人都是蠢货”的单向优越感。
- 女主暧昧、后宫、救美桥段。
- 现代政治口号直接落地古代。
- 大段论文式解释，尤其是连续出现“控制论、反馈、输入、输出、积分项”等术语。
- 全程苦大仇深。读者需要看到生活气、机锋、误会、笨拙试错和小胜利。

## 轻松感写法

推荐：

```text
严肃问题用具体小麻烦引出。
例如：主角想靠白糖暴富，结果第一锅糖糊了，系统只冷冷显示“焦化比例偏高”。
例如：主角想讲商业模式，鲁墨只关心锅为什么又裂了。
例如：税吏不知道肥皂算药、脂、香还是杂货，最后临时发明一个新税目。
```

不推荐：

```text
人物无脑卖蠢。
系统吐槽成段。
用网络段子破坏时代感。
把现代技术词硬塞进比喻，例如“中药系统写进人生主循环”。
```

## 草蛇灰线写法

伏笔要像生活细节，而不是显眼标记。

推荐形式：

- 一个短笑点：主角想提问题单。
- 一个小物件：缺口铁刃、裂锅、错账竹简。
- 一句闲话：葛婶说他烧糊涂了。
- 一个系统边角提示：某项损耗偏高。

回收方式：

- 后续用它解决一个问题。
- 后续让主角从它身上得到新的认知。
- 后续让读者发现早期笑点其实是主线入口。

每章不要刻意塞太多伏笔。每卷重点 1-3 条即可。

## 系统提示格式

系统提示必须短，像冷静账目。

可用：

```text
【未归之账：高】
【役损：三年内回流】
【信约折损：上升】
【兵偿不足】
```

避免：

```text
恭喜宿主触发隐藏任务！
当前社会控制系统反馈延迟严重，建议进行 PID 参数调节。
```
