# 任务：章节大纲

## 目的

把一个章节 brief 扩展成可写正文的分场景大纲。

## 输入

使用 `templates/chapter_brief.yaml`。

## 输出格式

```markdown
# 第 X 章：章名

## 本章功能

- 所处认知阶梯：
- 本章 target：
- 本章反馈/代价：
- 人物羁绊推进：
- 读者参与推导点：
- 若有衡仪/系统人格：出现阶段与功能：

## 场景拆分

### 场景 1
- 地点：
- 出场人物：
- 目标：
- 冲突：
- 信息增量：
- 情绪变化：

### 场景 2
...

## 本章结尾钩子

## 不能写的内容
```

## 检查点

- 本章至少一个具体物件或账目推动剧情。
- 本章必须让主角或局势发生一点变化。
- 本章不能只聊天。
- 若本章属于第一卷早中期，需写明它如何推进到“废庙与铁炉”。
- 若本章涉及管理、役棚、村社、铁炉、流民或衡山，需写明主角当前 target、反馈方式和失败代价。
- 若本章涉及现代小术赚钱，需写明钱如何提供试验资格，而不是直接解决秩序问题。
- 若本章涉及人物死亡或重伤，需写明此前羁绊是否足够、死亡是否来自系统缺陷和多重约束，而非临时煽情。
- 若本章涉及核心理论、假说、公理或推论，需写明前文哪些经历、现代记忆或草蛇灰线支撑它，且本章如何让读者先看见现象再听到命名。
- 若本章涉及系统人格，需检查是否达到出现阶段；衡仪只能追问、校验、转译，不能给最佳答案。
- 若本章涉及价值判断，需写明其价值来源、成本来源、利益去向和迟来反馈，不要只写“这是好事”。
- 若本章压力较重，应安排一个轻松互动、误会、冷淡系统提示或生活细节缓冲。
- 本章如使用草蛇灰线，需标明属于哪一条；不要新增无计划的大伏笔。
