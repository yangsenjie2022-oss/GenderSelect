# 任务：章节正文生成

## 目的

根据章节大纲生成可读正文。

## 输入

- `project.yaml`
- `../NOVEL_BLUEPRINT.md`
- `context/*`
- 本章大纲

## 输出要求

- 中文正文。
- 目标长度 3000-5000 字；若用户要求短样章，可降到 1500-2500 字。
- 不要在正文外解释理论。
- 章末可留悬念，但不要硬断。

## 写作策略

1. 先用具体场景开章。
2. 用人物动作和冲突带出信息。
3. 让系统提示只在关键处出现。
4. 主角的现代知识以短内心形式出现，不做长篇科普。
5. 每章结尾让读者想知道“这笔账后面会怎样回来”。
6. 第一卷早中期要把小技术、小买卖、小倒霉写得有趣，但每个笑点后面都应藏一个制度或成本问题。
7. 不要让章节过度苦大仇深。压力和轻松要交替。
8. 幽默要短而自然，避免生硬技术黑话比喻。
9. 若使用草蛇灰线，让它像自然细节，不要用旁白提醒“这是伏笔”。
10. 若章节涉及役棚、村社、铁炉或流民小邑，只推进对应层级的认知，不要提前写完整衡学。
11. 现代人权、平等、少受苦等直觉可以在早期作为主角真诚 target 出现，但必须通过具体反馈逐步修正为可运行制度。
12. 人物羁绊要靠小动作和小承诺累积；不要让角色只为承载理论或死亡而出现。
13. 诸子百家出场时要各有局部正确性，不能写成主角单方面嘲笑古人。
14. 核心理论第一次命名前，先写现象、冲突和失败；让读者先隐约推到，再由主角命名。
15. 系统/衡仪只问问题、校验口径、转译语言，不直接宣布结论或给最优解。
16. 中后期衡仪可带现代白月光师艺的回声，但不要写成实体女主、撒娇系统或恋爱主线。
17. 写价值判断时，必须追问成本从哪里来、利益往哪里走、迟来代价由谁承担。
18. 如果出现公理化推演，应让主角翻旧问题簿、回忆现代经历和人物代价，而不是凭空讲理论。

## 输出格式

```markdown
# 第 X 章：章名

正文...
```
