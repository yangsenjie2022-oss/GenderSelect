# 小说生成 Harness

这是《未归之账》（暂名）的 AI 小说生成工程。它不是单个提示词，而是一套可重复执行的写作流水线：读设定、规划章节、生成正文、检查连续性、修订风格、更新账本。

## 使用方式

每次让 AI 写作时，按下面顺序加载文件：

1. `project.yaml`
2. `context/00_harness_principles.md`
3. `../NOVEL_BLUEPRINT.md`
4. `context/*.md`，尤其是连续性账本、早中期收束、逐级顿悟框架、人物羁绊、系统人格、公理化推演、读者参与推导
5. 需要执行的 `tasks/*.md`
6. 对应的 `checks/*.md`
7. 本次章节输入模板，例如 `templates/chapter_brief.yaml`

推荐工作流：

```text
任务 1：用 tasks/01_arc_planner.md 细化卷纲
任务 2：用 tasks/02_chapter_outline.md 生成章节大纲
任务 3：用 tasks/03_chapter_draft.md 写章节正文
任务 4：用 checks/* 做质量审查
任务 5：用 tasks/04_chapter_rewrite.md 修订
任务 6：用 tasks/05_continuity_update.md 更新连续性账本
```

## 目录说明

```text
novel_harness/
  project.yaml                 全局配置和写作目标
  context/                     长期设定、风格、禁区、连续性
  tasks/                       AI 可执行任务
  checks/                      质量检查量表
  templates/                   每次运行的输入/输出模板
  runs/                        每次生成的实际记录，可手动保存
```

## 核心原则

- 写男性向穿越制度建设文，不写女主感情线。
- 系统只统计，不代打，不发奖励。
- 主角不能早期全知，必须从小术挫败走向一知半解。
- 主角的大理论必须经过四级顿悟：役棚、村社/里、县级工坊/铁炉、战时/灾年/迁民小邑。
- 现代物品复刻赚到的钱只能让主角买到试验资格，不能让他绕过组织、信用、制度和共同想象。
- 衡山从烂摊子和试验田中长出来，后续才成为衡学发源地。
- 人物羁绊是理论的内在动力。伍仓、陈简、许录、葛婶与孙子、公羊度等人应把制度问题变成具体牵挂。
- 中后期系统可形成“衡仪”人格，带有现代白月光林知微的认知回声；她是陪伴和校验，不是外挂女主。
- 公理化推演要像破案：公理与推论必须从前文经历、现代记忆和草蛇灰线中长出来。
- 读者要参与推导。先给现象和失败，再由主角命名；不要用作者旁白教育读者。
- 早中期的小技术、小买卖、小倒霉和小爽点，都要自然收束到“废庙与铁炉”这个第一次主题转折。
- 情节可以欢脱、机灵、有倒霉喜剧感；不要全程苦大仇深，也不要把正文写成论文。
- 每卷经营 1-3 条草蛇灰线：前期像笑点、小物件、闲话或小麻烦，后期回收时让读者有“原来如此”的感觉。
- 幽默来自职业错位、现实打脸和人物误会，避免把现代技术词硬套成生硬比喻。
- 控制论和社会理论只做底层骨架，正文尽量用时代语言表达。
- 爽点来自“看见未归之账”和“迟来的现实回流”，不是现代嘴炮碾压古人。
